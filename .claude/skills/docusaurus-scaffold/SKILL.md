## 1. Scaffold

```bash
# From repo root (not inside site/)
npx create-docusaurus@latest site classic --typescript
cd site
npm install
```

- `classic` preset with TypeScript is the baseline.
- Docusaurus lives in `site/` so its `docs/` never collides with `planning/`.

## 2. Enable Mermaid

Install the theme and configure it. Two things must both be present in `docusaurus.config.ts`:

```ts
// Top-level key alongside presets:
markdown: {
  mermaid: true,
},

themes: ['@docusaurus/theme-mermaid'],
```

Install the package:

```bash
npm install --save @docusaurus/theme-mermaid
```

Mermaid diagrams then render in light and dark mode natively — no plugin or remark config needed.

## 3. The `<Quiz>` React Component

Source: `site/src/components/Quiz/index.tsx` + `styles.module.css`

**Prop contract:**

```ts
type QuizOption   = { text: string; correct: boolean; explanation?: string };
type QuizQuestion = { prompt: string; options: QuizOption[]; multiSelect?: boolean };

export default function Quiz({ questions }: { questions: QuizQuestion[] }): JSX.Element
```

**Behaviour:**
- Single-select (default) or multi-select (`multiSelect: true`).
- Per-option instant feedback with explanation text on submit.
- Running score tally + reset button.
- No backend, no persistence — pure client React state.

**Usage in any quiz.mdx:**

```mdx
import Quiz from '@site/src/components/Quiz';

<Quiz questions={[
  {
    prompt: 'Question text?',
    multiSelect: true,
    options: [
      { text: 'Option A', correct: true,  explanation: 'Why correct.' },
      { text: 'Option B', correct: false, explanation: 'Why wrong.'  },
    ],
  },
]} />
```

The component is defined once in `site/src/components/Quiz/` and imported by every `quiz.mdx` across all modules. No per-module setup needed.

## 4. Module Layout (flat top-level)

Modules sit **flat** at `site/docs/` — no Day-1/Day-2 sub-directories:

```
site/docs/
├── intro.md
├── setup/
├── m1-container-native/
│   ├── lesson.md      (sidebar_position: 1)
│   ├── lab.md         (sidebar_position: 2)
│   └── quiz.mdx       (sidebar_position: 3)
├── m2-serving/
...
```

Day grouping lives only in `intro.md`'s program-at-a-glance table — not in the sidebar structure.

`sidebars.ts` uses `autogenerated` for the docs tree; each module folder becomes a category, ordered by `sidebar_position` frontmatter.

## 5. GitHub Pages Deploy Workflow

File: `.github/workflows/deploy.yml`

Key points:
- Trigger: push to `main` with paths matching `site/**` or the workflow file itself.
- Build runs from `working-directory: site` (`npm ci && npm run build`).
- Uses `actions/upload-pages-artifact@v3` + `actions/deploy-pages@v4`.
- `permissions: pages: write, id-token: write` required on the job.
- `concurrency: cancel-in-progress: false` — never cancel a production deploy mid-flight.

Repo settings: Pages → Source must be set to **GitHub Actions** (not a branch).

**Token gotcha:** pushing a new `.github/workflows/` file from the CLI requires a token with the `workflow` scope. A standard `repo`-scoped PAT will be rejected with `refusing to allow a OAuth App to create or update workflow`. Solution: generate a new PAT with `workflow` checked, or use a GitHub App token. Add the token to the shell session before pushing:

```bash
gh auth login   # re-auth and select workflow scope
# or
GITHUB_TOKEN=<token-with-workflow-scope> git push
```

## 6. docusaurus.config.ts Essentials

```ts
const config: Config = {
  url: 'https://<org>.github.io',
  baseUrl: '/<repo-name>/',
  organizationName: '<org>',
  projectName: '<repo-name>',

  onBrokenLinks: 'throw',   // catch dead links at build time

  markdown: { mermaid: true },
  themes: ['@docusaurus/theme-mermaid'],

  future: { v4: true },     // forward-compat flag

  presets: [['classic', {
    docs: { sidebarPath: './sidebars.ts' },
    blog: false,
  }]],
};
```

Set `blog: false` — this is a course, not a blog.

## 7. Verify Commands

```bash

npm --prefix site start

# Confirm Mermaid is installed
ls site/node_modules/@docusaurus/theme-mermaid 2>/dev/null && echo "OK"

# Check for broken admonitions (space form — renders literally)
grep -r ':::note \|:::tip \|:::info \|:::warning \|:::danger ' site/docs/
```

## 8. Environment Notes (arm64 Mac / Rancher Desktop)

- Node v22 / npm 10 — verified working.
- `docker` lives at `~/.rd/bin/docker`, not on default PATH. For automated calls: `PATH="$HOME/.rd/bin:$PATH" docker ...`
- Containers reach the native Ollama server at `http://host.docker.internal:11434` — never containerize Ollama on Apple Silicon.
- Lab RAM ceiling: 4–6 GB peak per module (16 GB machine is the reference).
