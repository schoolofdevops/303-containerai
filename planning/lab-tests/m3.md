**Model:** `HuggingFaceTB/SmolLM2-135M-Instruct` (float32), max-model-len 1024

## Five real incompatibilities found & fixed (reuse repo assumed Docker Desktop / x86)

1. **`range of CPUs is from 0.01 to 2.00`** — compose `cpus: 4.0` cap exceeded the VM's 2 CPUs.
   Fixed by allocating 4 CPUs / 6 GB to the runtime VM (documented as a lab setup step).
2. **`Too large swap space. 4.00 GiB out of the 3.82 GiB total CPU memory`** — vLLM `--swap-space`
   defaults to 4 GiB. Fixed with `--swap-space 1`.
3. **`numa_migrate_pages failed. errno: 1`** (EPERM) — CPU backend migrates NUMA pages at startup.
   Fixed with `cap_add: [SYS_NICE]` + `security_opt: ["seccomp:unconfined"]`.
4. **HTTP 500 `rms_norm_impl not implemented for 'BFloat16'`** — `--dtype auto` chose bf16, which CPU
   kernels can't run on arm64. Fixed with `--dtype float32`.
5. **float32 360M crash-looped** (KV cache didn't fit the 6 GB VM). Fixed by defaulting to the lighter
   **135M** model + `--max-model-len 1024` / `--max-num-seqs 4`.
