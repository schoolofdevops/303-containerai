## Task 1 — Lesson (analogies + ≥1 Mermaid)
Anatomy of a GenAI app: LLM endpoint + embedding model + vector DB + application. Analogy: a vector DB
is a **librarian who shelves books by meaning, not title** — you ask "how do I restart payments?" and she
hands you the right runbook even if it never says those exact words. The naive-RAG pipeline
(ingest→chunk→embed→retrieve→generate) — Mermaid of the flow with the container boundary (native Ollama
vs containerized Chroma+app). ChromaDB (lightest, default) with Qdrant/pgvector as scale-up. The
**"Learning Mode"** teaching move (visualize query embedding → similarity search → context → generation).
Where naive RAG breaks: weak queries, wrong chunks, no follow-up, no query rewriting — this motivates M6
(Agentic RAG). ~1000–1300 words.

## Task 2 — Lab (uses `labs/m5`)

service?" and see the grounded answer + Learning Mode pipeline view. Include the headless
`test_chroma`-style check as an optional CLI verification. Troubleshooting: Chroma client/server version
skew (why 0.5.20 is pinned), app can't reach Ollama (`host.docker.internal`, `OLLAMA_HOST=0.0.0.0`),
first embed is slow. Teardown `docker compose down`. What's next: naive RAG's limits → M6 Agentic RAG.
